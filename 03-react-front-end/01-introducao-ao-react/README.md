# Introdução ao React

## Pré-requisitos:

- Lógica de programação.
- HTML & CSS.
- JavaScript (Conhecimento básico de variáveis, funções, loops, condicionais, etc.)

## Bibliotecas & Frameworks: 

Uma biblioteca é como ir à sua loja de móveis favorita quando você já tem uma casa, você vai lá e compra só o que precisa.

Uma biblioteca é um conjunto de código pré-escrito economizando tempo e esforço nos seus projetos.

### Exemplos de bibliotecas
Efeitos sonoros para jogos, formatação de datas, formatação de dados (imagens, áudios, vídeos e etc), e as que são para desenvolvimento de aplicações front-end:

- React
- jQuery
- Bootstrap
- Tailwind CSS

### Por que devemos usar bibliotecas?

- Padrões de qualidade (usam padrões de mercado).
- Reutilização de código (você reutiliza o código da biblioteca).
- Eficiência no desenvolvimento (você gasta menos tempo desenvolvendo).
- Compatibilidade e manutenção (você reutiliza o código da biblioteca).
- Especialização e domínio de problemas (você reutiliza o código da biblioteca).

### Quais problemas podemos ter ao usarmos bibliotecas nos projetos?

- Tamanho do código (depende do tamanho da aplicação e da quantidade de bibliotecas utilizadas).
- Curva de aprendizado (depende da quantidade de bibliotecas e do quão complexas elas são).
- Dependência e problemas de break-changes (quando você atualiza a biblioteca, ela pode quebrar o funcionamento da aplicação).
- Risco de impacto negativo de desempenho (depende da quantidade de bibliotecas e do quão complexas elas são).
- Limitações de personalização (depende da biblioteca e do quão flexível ela é, o que pode te limitar em certos aspectos do projeto).

## Frameworks:

Um framework é como o modelo de uma casa: conjunto de planta-baixas e algumas escolhas limitadas em termos de arquitetura e design. Você não pode simplesmente mudar a estrutura básica, mas tem a liberdade de decorar os cômodos, escolher móveis e finalizar os acabamentos do seu jeito. O framework te dá uma direção e estrutura, mas permite personalização dentro desse padrão.

### Por que devemos usar frameworks?

- Produtividade no desenvolvimento (fornecem uma estrutura pronta).
- Padronização de código (usam padrões de mercado).
- Reutilização de código (você reutiliza o código do framework).
- Comunidade e suporte (comunidade ativa e documentação oficial).
- Integração com outras ferramentas (fornecem integrações com outras ferramentas).

### Exemplos de frameworks:

- Desenvolvimento front-end: Next.js, Angular, Vue.js, Ember.js, Svelte.js, Nuxt.js.
- Desenvolvimento back-end: Express.js, Koa.js, Nest.js, Sails.js, Meteor.js, hapi.js.
- Desenvolvimento de aplicativos mobile: React Native, Flutter, Ionic, Xamarin, NativeScript, Cordova.
- Desenvolvimento de jogos: Phaser, PixiJS, Babylon.js, Three.js, PlayCanvas, Godot Engine.

## Podemos considerar o react um...

Uma biblioteca focada apenas na interface do usuário (UI), você consegue construir sites institucionais, portfólios, e-commerces e blogs com o React. Além disso, você consegue integrar o React com outras bibliotecas para adicionar funcionalidades extras, como:

- Roteamento (React Router).
- Gerenciamento de estado (Redux, Zustand, Recoil, MobX).
- Componentes de interface (Material UI, Bootstrap, Tailwind CSS).
- Animações (Framer Motion, React Spring).
- Testes (Jest, React Testing Library).
